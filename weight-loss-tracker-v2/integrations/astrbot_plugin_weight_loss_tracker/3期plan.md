# 第 3 期：P6.1 AstrBot 快速记录

## 范围

- 饮食和运动在同一次 LLM tool 调用中自动完成后端预览和正式写入
- 热量计划和撤销继续使用 `weight_confirm`
- Web 人工确认、后端 API、数据库和配置保持不变
- 自动写入失败时不创建待确认项，用户重新发送原记录即可

## 交付物

- `core/service.py`：饮食和运动 preview 指纹取得后立即创建并返回实际预算
- `main.py`：更新工具说明，插件版本升级为 v0.2.1
- 更新单测、README、总计划和 P6 当前行为文档

## 验证

- LLM 估算和用户明确提供的食物均一次写入
- 运动一次写入，结果包含最新预算
- 同一消息重试不重复创建记录
- 写入失败不留下食物或运动待确认项
- 热量计划、撤销和 QQ 隔离行为不变
- AstrBot 运行副本加载 v0.2.1，并通过真实后端自动记录和清理 smoke test
